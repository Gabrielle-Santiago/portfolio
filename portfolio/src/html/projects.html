Agenda Automática

Conceito

A aplicação foi desenvolvida para a empresa Espaço Bela Berillo (empresa real) e tem como objetivo automatizar os processos de agendamento e solicitação de profissionais especializados em design de sobrancelhas.

O projeto conta com uma tela de cadastro, onde o cliente agenda um horário e escolhe um procedimento estético.

As seguintes restrições são aplicadas:

  • Não é possível registrar horários iguais no mesmo dia.
  • Não é permitido agendar procedimentos cujo tempo ultrapasse o horário de expediente ou caia durante o intervalo.
  • Alguns procedimentos exigem uma avaliação prévia obrigatória antes do agendamento.
  • Validações básicas como e-mail no formato correto, nome com pelo menos 3 letras, número de telefone válido e data no padrão brasileiro.

Caso todas as validações sejam atendidas e o agendamento confirmado, o cliente recebe um e-mail com as principais informações: procedimento escolhido, data e horário, além de orientações como não levar crianças pequenas, evitar objetos cortantes e formas de pagamento aceitas. O proprietário visualiza a lista de agendamentos registrados, com a opção de deletar clientes cadastrados.

Além do agendamento convencional, há uma aba para solicitar produtos caseiros oferecidos pela empresa.

Nessa tela, o cliente informa nome, e-mail, telefone, produto, quantidade e aroma. Após o pedido, um e-mail de confirmação é enviado ao cliente, informando que a produção se inicia após o pagamento de 50% do valor, incluindo informações sobre taxas. O proprietário recebe detalhes sobre o pedido e os dados do cliente para possível contato.

OBS: Para acessar a lista de pedidos e agendamentos, o proprietário deve realizar login com usuário e senha previamente cadastrados.

Informações Técnicas

A aplicação foi desenvolvida em Python, utilizando o framework Django, com PostgreSQL como banco de dados e Django Mail para envio de e-mails.

O front-end foi organizado com JavaScript, Flatpickr (para configuração dos horários), Bootstrap e CSS, proporcionando um design simples e responsivo.

Para mais detalhes técnicos, acesse: GitHub Agenda Automática

GameCatalog

Uma aplicação que consome a API externa OpenCritic. Seu objetivo é apresentar informações e avaliações de jogos.

A interface conta com cartas contendo botões para avançar, retornar e visualizar mais informações. Na tela principal, são exibidos imagem do jogo, nota geral da crítica e ano de lançamento.

Ao clicar em “Mais informações”, o usuário visualiza nome, descrição, porcentagem de recomendação, quantidade de avaliações, média geral, gênero, plataformas disponíveis e o link oficial no OpenCritic.

Informações Técnicas

A aplicação foi desenvolvida em Java com Spring Boot, estruturada por rotas para consumo adequado da API.

O front-end foi construído com React e CSS, hospedado em repositório separado.

Para mais informações técnicas: GitHub GameCatalog

System Delivery

Simula um sistema de pedidos de alimentos, onde o usuário pode se cadastrar como vendedor ou consumidor.

Vendedor

Permite cadastrar, editar e excluir produtos, que devem conter nome, valor, descrição e imagem.

Cliente

O cliente visualiza os produtos e pode simular uma compra, cadastrando um cartão com validações padrão (número de dígitos e bandeira).

Após a compra, o vendedor recebe um e-mail notificando o pedido.

Tela Principal

A primeira tela orienta o usuário sobre o funcionamento da aplicação e suas vantagens, utilizando imagens e textos curtos, de forma intuitiva, seguindo o padrão de apps reais.

Informações Técnicas

Desenvolvido em Java com Spring Boot, Java Mail e PostgreSQL como banco de dados.

O front-end utiliza Thymeleaf, CSS e JavaScript, com design simples e agradável.

Para mais informações técnicas: GitHub System Delivery

Aprendizados

A cada projeto, os conhecimentos técnicos e boas práticas foram aprimorados. Entre os principais aprendizados estão:

  • Tratamento de erros: maior eficiência na identificação e resolução de erros, prevenindo falhas em tempo de execução.
  • Boas práticas de código: aplicação de conceitos como Clean Code, código desacoplado, versionamento seguro (.env ou .properties protegidos), e organização clara de arquivos.
  • Separação de responsabilidades: melhor compreensão da importância de dividir front-end e back-end em repositórios distintos, facilitando manutenção e escalabilidade.
  • Integração com APIs: aprofundamento no consumo de APIs externas, validação de dados recebidos e tratamento de respostas.
  • Documentação e versionamento: utilização de ferramentas como GitHub e README estruturados, descrevendo ambiente, instalação e uso das aplicações.
  • Pesquisa direcionada: valorização da escolha adequada de frameworks, bibliotecas e APIs, considerando simplicidade, desempenho e compatibilidade.
  • Responsividade e usabilidade: preocupação constante com a experiência do usuário e compatibilidade em diferentes dispositivos.
